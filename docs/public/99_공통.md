# 99. 공통 기능 (common 패키지)

> `com.tmi.core.common` 아래에 두는 공통 클래스 카탈로그.

---

## 1. 패키지 구조

```
com.tmi.core.common
├── config/
│   ├── JpaAuditingConfig
│   └── RestClientConfig
├── domain/
│   └── BaseEntity
└── exception/
    ├── IErrorType
    ├── ETmiErrorType
    ├── BaseException
    ├── GlobalExceptionHandler
    └── ErrorResponse
```

ErrorType 은 **모듈 단일 enum** (`ETmiErrorType`) — 공통·도메인 에러를 한 enum 에 통합. 도메인 예외 클래스는 도메인 패키지에 (예: `sentence/exception/SentenceException`) — 외부 클래스 + 정적 중첩 클래스 패턴.

---

## 2. 클래스

### 2.1 BaseEntity

**위치**: `common/domain/BaseEntity.java`

모든 엔티티의 공통 컬럼.

| 항목 | 내용 |
|---|---|
| 종류 | `abstract class` |
| 어노테이션 | `@MappedSuperclass`, `@Getter`, `@EntityListeners(AuditingEntityListener.class)` |
| 필드 | `createdAt`, `updatedAt`, `deletedAt` (`Instant`) |
| 자동 갱신 | `createdAt` = `@CreatedDate`, `updatedAt` = `@LastModifiedDate` |
| `deletedAt` | 자동 갱신 X. 서비스에서 명시적으로 `now()` 세팅 |

**컬럼 매핑**

| 필드 | 컬럼 | 옵션 |
|---|---|---|
| `createdAt` | `created_at` | `nullable=false`, `updatable=false` |
| `updatedAt` | `updated_at` | `nullable=false` |
| `deletedAt` | `deleted_at` | `nullable=true` |

id 는 두지 않음. 각 엔티티가 본인 PK 직접 선언.

### 2.2 JpaAuditingConfig

**위치**: `common/config/JpaAuditingConfig.java`

Spring Data Auditing 활성화 — `@CreatedDate` · `@LastModifiedDate` 어노테이션이 동작하도록 스위치만 켜는 설정.

| 항목 | 내용 |
|---|---|
| 어노테이션 | `@Configuration`, `@EnableJpaAuditing` |
| 본문 | 빈 클래스 |

### 2.3 RestClientConfig

**위치**: `common/config/RestClientConfig.java`

외부 시스템 호출용 `RestClient` + HTTP Interface (`@HttpExchange`) 기반 Proxy Factory 등록.

| 항목 | 내용 |
|---|---|
| 어노테이션 | `@Configuration` |
| 구성 | `RestClient` 빈 (타임아웃 명시) + `RestClientAdapter` + `HttpServiceProxyFactory` |
| 공통 설정 | connect timeout 1s, read timeout 2s |
| 클라이언트 등록 | 외부 시스템마다 `HttpServiceProxyFactory.createClient(XxxClient.class)` 로 빈 등록 |

개별 client 인터페이스 (예: `SimilarityClient` — `@HttpExchange` 인터페이스) 는 여기 두지 않음. `infra/{시스템명}/` 에 인터페이스로 선언, 본 Config 에서 빈 등록.

→ OpenFeign 미채택. Spring 6+ 표준인 HTTP Interface + RestClient 조합 채용.

### 2.4 IErrorType

**위치**: `common/exception/IErrorType.java`

모든 ErrorType enum 의 공통 계약.

| 항목 | 내용 |
|---|---|
| 종류 | `interface` |
| 메서드 | `String getErrorCode()` |

HttpStatus 는 enum 에 묶지 않음 — 예외 생성 시점에 명시 (`BaseException` 생성자 인자).

### 2.5 ETmiErrorType

**위치**: `common/exception/ETmiErrorType.java`

**모듈 단일 enum** — 공통 + 모든 도메인 에러를 한 enum 에 통합.

| 항목 | 내용 |
|---|---|
| 종류 | `enum implements IErrorType` |
| 어노테이션 | `@RequiredArgsConstructor`, `@Getter` |
| 필드 | `String errorCode` |

| 영역 | 상수 | 코드 |
|---|---|---|
| 공통 | `INVALID_REQUEST` | `TMI.COMMON.INVALID_REQUEST` |
| 공통 | `INTERNAL_ERROR` | `TMI.COMMON.INTERNAL_ERROR` |
| 공통 | `NOT_FOUND_URL` | `TMI.COMMON.NOT_FOUND_URL` |
| 공통 | `METHOD_NOT_SUPPORTED` | `TMI.COMMON.METHOD_NOT_SUPPORTED` |
| 공통 | `MISSING_PARAMETER` | `TMI.COMMON.MISSING_PARAMETER` |
| 공통 | `BAD_REQUEST` | `TMI.COMMON.BAD_REQUEST` |
| Sentence | `SENTENCE_NOT_FOUND` | `TMI.SENTENCE.NOT_FOUND` |
| Similarity | `SIMILARITY_REQUEST_INVALID` | `TMI.SIMILARITY.REQUEST_INVALID` |
| Similarity | `SIMILARITY_UNAVAILABLE` | `TMI.SIMILARITY.UNAVAILABLE` |

새 도메인 추가 시 enum 상수만 추가. 도메인별 enum 으로 분리하지 않음.

### 2.6 BaseException

**위치**: `common/exception/BaseException.java`

모든 도메인 예외의 추상 부모.

| 항목 | 내용 |
|---|---|
| 종류 | `abstract class extends RuntimeException` |
| 필드 | `HttpStatus httpStatus`, `IErrorType errorType`, `Object[] args`, `ELogLevel logLevel`, `String logMessage` |
| 생성자 | 4 종 — `(HttpStatus, IErrorType)` · `(HttpStatus, IErrorType, Throwable)` · `(HttpStatus, IErrorType, Object...)` · `(HttpStatus, IErrorType, Throwable, Object...)` |
| HTTP 상태 | `getHttpStatus()` 필드 그대로 반환 |
| chaining | `logLevel(...)`, `logMessage(...)` setter 가 자식 타입 그대로 반환 (제네릭) |

도메인 예외는 **외부 클래스 + 정적 중첩 클래스** 패턴 (예: `SentenceException.NotFound`). 외부 클래스는 인스턴스화 금지 (`private` 생성자), 케이스마다 정적 중첩 클래스가 `BaseException` 직접 상속.

사용:
```
throw new SentenceException.NotFound(sentenceId);
```

### 2.7 GlobalExceptionHandler

**위치**: `common/exception/GlobalExceptionHandler.java`

컨트롤러에서 발생한 예외를 가로채 표준 에러 응답으로 변환하는 단일 핸들러.

| 항목 | 내용 |
|---|---|
| 어노테이션 | `@RestControllerAdvice` |
| 처리 대상 | `BaseException` · `MethodArgumentNotValidException` (body validation) · `MethodArgumentTypeMismatchException` (parameter 타입 변환 실패, enum 등) · `NoResourceFoundException` (404 URL) · `HttpRequestMethodNotSupportedException` (405) · `MissingServletRequestParameterException` · `Exception` (catch-all) |
| 응답 | `ErrorResponse` + `e.getHttpStatus()` 직접 사용 |

`@RestControllerAdvice` 는 본 클래스 하나만. 도메인별 advice 추가 금지.

### 2.8 ErrorResponse

**위치**: `common/exception/ErrorResponse.java`

클라이언트로 나가는 에러 JSON 구조의 DTO.

```json
{ "error": { "code": "TMI.SENTENCE.NOT_FOUND", "message": "Sentence abc-123 was not found." } }
```

| 항목 | 내용 |
|---|---|
| 종류 | `record` 또는 일반 클래스 (Lombok `@Getter @Builder` 가능) |
| 필드 | `Error error` → `code`, `message` |

## 3. 메시지

**위치**: `src/main/resources/messages/messages.properties`, `messages_ko.properties`

| 파일 | 언어 |
|---|---|
| `messages.properties` | 영문 (기본) |

```properties
TMI.COMMON.INVALID_REQUEST = Invalid request.
TMI.COMMON.INTERNAL_ERROR = Internal server error.
TMI.SENTENCE.NOT_FOUND = Sentence {0} was not found.
TMI.SIMILARITY.REQUEST_INVALID = Invalid request to similarity service.
TMI.SIMILARITY.UNAVAILABLE = Similarity scoring service is unavailable.
```

키 = ErrorType enum 의 `errorCode` 값.
