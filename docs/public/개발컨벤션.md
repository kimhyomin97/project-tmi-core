# 개발 컨벤션

> TMI Core BE 개발 시 따르는 규칙. 필요한 항목을 한 섹션씩 추가.

---

## 1. 프로젝트 / 패키지 구조

### 1.1 아키텍처

계층형 (Controller / Service / Repository).

### 1.2 루트 패키지

`com.tmi.core`

### 1.3 패키지 분할

도메인 우선 — 도메인별로 먼저 분할, 안에서 레이어로 분할.

```
com.tmi.core
├── CoreApplication                 ← @SpringBootApplication
├── {도메인}/                        ← 도메인 단위 (quiz, ...)
│   ├── controller/
│   ├── service/
│   ├── repository/
│   ├── domain/                     ← 엔티티 + 값 객체
│   └── dto/
│       ├── request/
│       └── response/
├── infra/{외부시스템}/              ← Similarity Server 등 외부 클라이언트
└── common/                         ← Spring 설정, 상수/enum, 예외
    ├── config/
    ├── define/
    └── exception/
```

### 1.4 도메인 목록 (Phase 1)

| 도메인 | 책임 |
|---|---|
| `quiz` | 영어 문장 출제 / 답안 채점 / 유사도 서버 호출 |

이력·통계·사용자 등은 v0.2+ 단계에서 별도 도메인으로 분리 (요구사항 F-05, F-07).

## 2. 네이밍

### 2.1 기본

표준 Java 코드 스타일(Google Java Style 또는 IntelliJ 기본)을 따른다. 들여쓰기 4 spaces, import wildcard 금지, 메서드/변수 camelCase.

### 2.2 계층 Suffix

| 레이어 | Suffix | 예시 |
|---|---|---|
| Controller | `Controller` | `QuizController` |
| Service | `Service` | `QuizService` |
| Repository | `Repository` | `SentenceRepository` |

### 2.3 DTO

- Request: `{Verb}{Domain}Request` 또는 `{Domain}{용도}Request` (예: `QuizAnswerRequest`)
- Response: `{Domain}Response` 또는 `{Domain}{용도}Response` (예: `QuizQuestionResponse`, `QuizResultResponse`)

### 2.4 변수·컬럼

- Java 변수·필드: camelCase (`sentenceId`)
- DB 컬럼: snake_case (`sentence_id`) — §3.1

## 3. DB 스키마

### 3.1 테이블·컬럼 명명

- `snake_case`
- 테이블명은 **단수형** (`sentence`, `attempt`)
- FK 컬럼은 `{참조 테이블}_id` (`sentence_id`)

### 3.2 PK / ID

- PK는 단일 컬럼 `id`, 타입 `varchar(36)` (순수 UUID)
- 서비스 레이어에서 `UUID.randomUUID().toString()`으로 생성 (DB default 미사용)
- **예외**: 카탈로그성 enum 값(예: `difficulty`, `category`)은 의미 있는 식별자 (`STARTER`, `DAILY`)

### 3.3 타입

| 용도 | 타입 |
|---|---|
| ID | `varchar(36)` |
| 짧은 문자열 | `varchar(N)` — 길이 명시 |
| 긴 문자열 (문장 본문 등) | `text` |
| 시각 | `timestamptz` (UTC). `timestamp` 사용 금지 |
| 정수 | `int` / `bigint` |
| 소수 (유사도 점수 등) | `numeric`. `float`/`double` 사용 금지 |
| 가변 구조 | `jsonb` |

### 3.4 enum

- PostgreSQL `enum` 타입 사용 금지
- **`varchar(N) + CHECK constraint`** 패턴

```sql
difficulty varchar(20) NOT NULL
  CHECK (difficulty IN ('STARTER', 'BEGINNER', 'INTERMEDIATE', 'ADVANCED', 'CHALLENGE'))
```

- JPA 매핑은 `@Enumerated(EnumType.STRING)` + 같은 문자열

### 3.5 공통 컬럼

사용자 자원 테이블(`sentence` 등)에 적용.

| 컬럼 | 타입 | 비고 |
|---|---|---|
| `created_at` | `timestamptz NOT NULL DEFAULT now()` | |
| `updated_at` | `timestamptz NOT NULL DEFAULT now()` | |
| `deleted_at` | `timestamptz NULL` | NULL이면 활성 (soft delete) |

- 이력(`attempt`): `created_at`만 적용 — 사용자가 수정·삭제할 수 없는 append-only 감사 영역

### 3.6 Soft Delete

- 사용자 자원 테이블에 적용. `deleted_at IS NULL`이면 활성.
- 조회 쿼리는 항상 `WHERE deleted_at IS NULL`.
- 예외: **이력 테이블** (`attempt`) — 사용자가 수정·삭제할 수 없는 append-only 감사 영역

## 4. API 컨벤션

### 4.1 URL

- 버저닝: `/v{n}/...` (`application.yml`의 `rest.api.version` 치환)
- 자원 명: **복수형 + kebab-case** (`/sentences`, `/attempts`)
- URL에 동사 금지. 동사형 액션은 **변경 대상**(상태/속성)을 sub-path로 표현
  - ❌ `POST /attempts/{id}/finalize`
  - ✅ `PATCH /attempts/{id}/status` (status 변경)
  - ✅ `PUT /users/{id}/password` (password 변경)

### 4.2 HTTP 메서드 ↔ 응답 코드

| 메서드 | 용도 | 응답 |
|---|---|---|
| GET | 조회 | 200 |
| POST | 생성 | 201 + `Location` 헤더 |
| PUT | 전체 수정 | 200 |
| PATCH | 부분 변경 | 200 |
| DELETE | 삭제 | 204 |

### 4.3 페이징

커서 기반. `limit` 기본 20 / 최대 100.
응답: `{ items: [...], page: { cursor, nextCursor, limit, hasMore } }`

### 4.4 시간 형식

ISO 8601 UTC (`2026-05-08T10:00:00Z`).

## 5. 에러 처리

### 5.1 베이스

Spring의 `@RestControllerAdvice` 기반 글로벌 핸들러로 일원화. 응답 포맷·핸들러·다국어 메시지는 자체 컴포넌트로 구성한다.

작성 대상은 4가지:

| 작성할 것 | 위치 | 역할 |
|---|---|---|
| `ETmiErrorType` enum | `common/exception/` | 에러 코드 카탈로그 |
| `{Reason}Exception` 클래스 | `common/exception/` 또는 도메인 패키지 | 각 에러를 던질 때 쓰는 예외 |
| `GlobalExceptionHandler` | `common/exception/` | `@RestControllerAdvice` 핸들러 |
| `messages*.properties` | `src/main/resources/messages/` | 코드별 사용자 메시지 |

### 5.2 응답 포맷

```
HTTP 404
{
  "error": {
    "code": "TMI.SENTENCE.NOT_FOUND",
    "message": "Sentence abc-123 was not found."
  }
}
```

- 래핑 구조 `{ "error": { code, message } }` — `GlobalExceptionHandler`가 `ErrorResponse` DTO로 직렬화
- `code` 형식: **`TMI.{DOMAIN}.{REASON}`**
- `message`: 영문 기본. 다국어는 `messages_{locale}.properties`로 추가

### 5.3 ErrorType enum

- 자체 `IErrorType` 인터페이스(또는 마커 인터페이스) 구현
- enum 상수 1개 = 에러 1종

```java
@RequiredArgsConstructor
@Getter
public enum ETmiErrorType implements IErrorType {
    SENTENCE_NOT_FOUND("TMI.SENTENCE.NOT_FOUND"),
    ATTEMPT_NOT_FOUND("TMI.ATTEMPT.NOT_FOUND"),
    SIMILARITY_UNAVAILABLE("TMI.SCORING.UNAVAILABLE"),
    ;

    private final String errorCode;
}
```

### 5.4 예외 클래스

- 추상 `BaseException`(`RuntimeException` 상속) 정의 후 도메인 예외가 이를 상속
- 1 에러 = 1 클래스 = 1 파일
- 생성자에서 `HttpStatus` + `IErrorType` + 메시지 치환 인자 전달

```java
public class SentenceNotFoundException extends BaseException {
    public SentenceNotFoundException(String sentenceId) {
        super(HttpStatus.NOT_FOUND, ETmiErrorType.SENTENCE_NOT_FOUND, sentenceId);
    }
}
```

서비스에서 `throw new SentenceNotFoundException(sentenceId)` 호출 시 `GlobalExceptionHandler`가 응답까지 처리한다. **Controller에서 try/catch 금지**, **추가 `@RestControllerAdvice` 작성 금지** (핸들러 단일화).

### 5.5 메시지

- `messages.properties`: 영문 (기본)
- `messages_ko.properties`: 한글
- `{0}`, `{1}`은 예외 생성자의 가변 인자 순서대로 치환

```properties
# messages.properties
TMI.SENTENCE.NOT_FOUND = Sentence {0} was not found.
TMI.ATTEMPT.NOT_FOUND = Attempt {0} was not found.
TMI.SCORING.UNAVAILABLE = Similarity scoring service is unavailable.
```

### 5.6 HTTP status 선택 기준

| 상황 | status |
|---|---|
| 잘못된 요청 (validation 실패, 형식 오류) | `400 Bad Request` |
| 인증 실패 | `401 Unauthorized` |
| 권한 없음 | `403 Forbidden` |
| 리소스 미존재 | `404 Not Found` |
| 상태 충돌 (중복 / 사용 중 / 한도 초과) | `409 Conflict` |
| 서버 내부 오류 | `500 Internal Server Error` |
| 외부 시스템 호출 실패 (Similarity Server) | `502 Bad Gateway` 또는 `503 Service Unavailable` |

### 5.7 외부 시스템 예외 매핑

Similarity Server(Python) 호출 시 발생하는 HTTP 클라이언트 예외는 TMI 도메인 예외로 변환 후 throw한다.

```java
try {
    return similarityClient.score(request);
} catch (HttpClientErrorException e) {
    throw new SimilarityRequestInvalidException(e);
} catch (ResourceAccessException | HttpServerErrorException e) {
    throw new SimilarityUnavailableException(e);
}
```

raw HTTP 클라이언트 예외를 그대로 전파 금지 (외부 응답 형식의 사용자 노출 방지).

## 6. 로깅

### 6.1 베이스

Spring Boot 기본 SLF4J + Logback 사용. 분산 추적이 필요해지면 Micrometer Tracing(Brave / OpenTelemetry) 의존성 추가로 `traceId` / `spanId` 자동 주입.

### 6.2 기본 설정

| 영역 | 설정 |
|---|---|
| 로그 라이브러리 | SLF4J + Logback (Spring Boot starter 기본) |
| 콘솔 포맷 | Spring Boot 기본 (timestamp / level / thread / logger / msg) |
| 파일 포맷 | 필요 시 `logback-spring.xml`에서 JSON encoder(예: logstash-logback-encoder) 적용 |
| 파일 회전 | 필요 시 `RollingFileAppender` + 일일 rotate + gzip |
| 분산 추적 MDC | `traceId`, `spanId` (Micrometer Tracing 활성화 시 자동) |
| 민감정보 마스킹 | 별도 라이브러리 미사용. 코드에서 토큰·비밀번호를 출력하지 않는 규칙으로 통제 |

### 6.3 프로젝트 설정 (`application.yml`)

```yaml
logging:
  level:
    root: INFO
    com.tmi.core: DEBUG   # 운영은 INFO
```

### 6.4 로거 선언

- `@Slf4j` (Lombok) 사용
- `LoggerFactory.getLogger()` 직접 호출 금지

```java
@Slf4j
@Service
public class QuizService { ... }
```

### 6.5 메시지 작성 규칙

**작성:**
```java
log.info("문제 출제 요청: difficulty={} sentenceId={}", difficulty, sentenceId);
log.error("유사도 서버 호출 실패: sentenceId={}", sentenceId, e);
```

**출력 (Spring Boot 기본 포맷 예):**
```
2026-05-26T10:00:00.000+09:00  INFO 12345 --- [http-nio-8080-exec-3] c.t.c.quiz.service.QuizService           : 문제 출제 요청: difficulty=INTERMEDIATE sentenceId=abc-123
2026-05-26T10:00:01.000+09:00 ERROR 12345 --- [http-nio-8080-exec-3] c.t.c.quiz.service.QuizService           : 유사도 서버 호출 실패: sentenceId=abc-123
org.springframework.web.client.ResourceAccessException: I/O error on POST request for "http://similarity/score": Connection refused
    at ...
```

- 변수 출력은 placeholder `{}` 사용 — 문자열 `+` 연결 금지
- ERROR 로그는 `Throwable`을 마지막 인자로 전달 → stacktrace 자동 부가
- 메시지 언어는 한글·영문 자유

### 6.6 로그 레벨 사용 기준

| 레벨 | 용도 |
|---|---|
| `INFO` | 주요 비즈니스 이벤트 (출제, 채점 시작·완료, 외부 호출 시작·완료) |
| `DEBUG` | 상세 파라미터·반환값 추적 (운영 비활성) |
| `WARN` | 예외 복구 시나리오, 보안 이벤트 (인증 실패 등) |
| `ERROR` | 예외 발생 — 스택트레이스 필수 (`Throwable` 마지막 인자) |

### 6.7 금지

- 민감정보 raw 로깅 (인증 토큰, API 키, 비밀번호, 외부 시크릿)
- 운영 환경 DEBUG 활성화
