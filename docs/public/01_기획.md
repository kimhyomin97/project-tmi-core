# 01. 기획

> TMI Core가 풀려는 문제, 사용자 경험, 만들 기능의 정의.

**대상 독자**: 프로젝트에 처음 합류하거나 외부에서 둘러보는 개발자  
**범위**: 무엇을·왜 만드는가  
**관련 문서**: [02. 설계](./02_설계.md) — 어떻게 만드는가

## 1. 배경

### 1.1 문제 인식

영어 학습 시장에 단어 암기·문법 드릴·문장 듣기 도구는 풍부함. 그러나 **"영어 문장을 한국어로 옮기는 번역 연습"** 영역은 상대적으로 도구가 적음.

번역 연습에는 두 가지 어려움이 있음:

- **정답이 하나가 아님** — 같은 영어 문장도 한국어로는 여러 자연스러운 표현으로 옮길 수 있음.
- **자동 채점이 까다로움** — 문자열 일치(Levenshtein 등)만 보면 "맞는 답인데 점수 0"이 빈번.

### 1.2 해결 방향

문자열 매칭 대신 **의미적 유사도(semantic similarity)** 기반 채점.
- 모범 답안과 사용자 답안을 각각 벡터로 임베딩.
- 두 벡터의 코사인 유사도를 점수로 환산.
- 표현이 달라도 의미가 가까우면 높은 점수.

### 1.3 차별점

| 일반 영어 학습 앱 | TMI |
|---|---|
| 객관식·빈칸·OX | 자유 번역 입력 |
| 정답 1개 매칭 | 의미 유사도 채점 |
| 범용 콘텐츠 | 개인 학습용 (필요 영역만 출제) |

---

## 2. 사용자 시나리오

### 2.1 페르소나

- **본인 = 단일 사용자** (개인 학습 도구).
- 영어 독해는 가능, 번역 표현력을 키우고 싶은 학습자.
- 짧은 자투리 시간(5~15분)에 한 번씩 사용.

### 2.2 대표 시나리오 — 자투리 학습

> 출퇴근 지하철에서 휴대폰을 열어 난이도 `INTERMEDIATE`, 카테고리 `DAILY`로 출제.
> 영어 문장을 보고 머릿속으로 한국어로 옮긴 뒤 입력 → 채점 결과로 모범 답안과 점수가 즉시 표시.
> 다음 문제로 이어 5~10문제를 풀고 종료.

## 3. 핵심 사이클

사용자 한 번의 학습 루프 = **출제 → 입력 → 채점 → 피드백** 4단계 순환.

📊 **[학습 사이클 다이어그램 (HTML)](./assets/diagrams/quiz-cycle.html)**

| # | 단계 | 주체 | 산출 |
|---|---|---|---|
| 1 | 출제 | System → User | 영어 문장 (모범 답안 숨김) |
| 2 | 번역 입력 | User → System | 한국어 답안 |
| 3 | 채점 | System (AI) | 유사도 점수 |
| 4 | 피드백 | System → User | 점수 + 점수 등급 + 내 답 ↔ 모범 답안 병렬 표시 |

→ 4 이후 다음 문제로 이어지며 사용자가 종료할 때까지 반복.

> **§4 피드백 = 학습 도움의 핵심.** 점수만으로는 학습 효과가 약함. 점수의 의미를 직관적으로 알 수 있는 등급 라벨, 그리고 내 답과 모범 답안의 병렬 표시 → 차이를 인식하고 다음 시도에 반영.

## 4. 기능 목록

### 4.1 MVP 기능

학습 사이클 1바퀴를 끝까지 완결하는 데 필요한 최소 기능.

| ID | 기능 | 설명 |
|---|---|---|
| F-01 | 영어 문장 출제 | 난이도·카테고리 필터로 1건 랜덤 추출. 모범 답안은 숨김 |
| F-02 | 한국어 번역 입력 | 사용자 자유 입력, 길이 상한만 둠 |
| F-03 | 의미 유사도 채점 | 모범 답안과 사용자 답안의 의미 유사도 → 0.0 ~ 1.0 점수 |
| F-04 | 결과 피드백 | 점수 + 점수 등급(우수/양호/보통/미흡) + **내 답 ↔ 모범 답안 병렬 표시** |
| F-05 | 다음 문제 진행 | 동일 필터로 즉시 새 문제 출제, 학습 루프 유지 |

### 4.2 MVP 이후 검토 사항

MVP를 일정 기간 사용해본 뒤, 다음 후보들의 도입 여부·우선순위 결정. 단계 분할은 두지 않음 — 실사용 신호가 모이면 그때 결정.

- **답안 이력 저장** → 본인 점수 변화 추적, 약점 영역 자동 인식, 약점 우선 출제
- **채점 시각화 강화** → 단어 단위 비교 하이라이트, 자연스러운 다른 번역 표현 제시
- **학습 모드 확장** → 한→영 방향, 단어 학습
- **멀티 사용자** → 회원가입·로그인 (현재 단일 사용자 가정)

→ 본 문서는 각 기능이 **무엇·왜**까지만 다룸. **어떻게**는 [02. 설계](./02_설계.md).

## 5. 다루지 않는 것 (Non-Goals)

| 제외 항목 | 이유 |
|---|---|
| TTS·발음 평가 | 읽기·쓰기 학습이 우선, 오디오 영역은 별 도구가 풍부 |
| 통계 대시보드 (시각 차트) | 이력이 충분히 쌓이기 전엔 의미 없음, v0.2 이후 재검토 |
| 소셜·랭킹·공유 | 개인 학습 도구 정체성과 어긋남 |
| 강의·해설 콘텐츠 | 채점·피드백 도구에 집중 |
| 모바일 네이티브 앱 | 반응형 웹으로 충분 (PWA는 v1.0+ 고려) |

## 6. 난이도 체계

영어 문장의 **단어 수**를 기준으로 5레벨 분류. 숫자 + 의미 라벨 병기.

| 레벨 | 라벨 | 단어 수 | 성격 | 예시 |
|---|---|---|---|---|
| **1** | 입문 | 1 ~ 3 | 단편 표현 | "Sounds good." |
| **2** | 기초 | 4 ~ 8 | 짧은 문장 | "What did you do yesterday?" |
| **3** | 중급 | 9 ~ 15 | 일상 회화 | "I had a rough day at work today." |
| **4** | 고급 | 16 ~ 25 | 복문·종속절 | "Although the weather forecast warned of rain, we decided to go ahead with the picnic." |
| **5** | 도전 | 26 ~ 35 | 긴 단락·복합 | "Even though the deadline had passed and most of the team had already moved on to other tasks, she insisted on revisiting the proposal because she still saw untapped opportunities." |

> 단어 수 기준 = 객관적 측정, 별도 라벨링 부담 없음.
> 숫자만 보고도 난이도 순서가 즉시 파악되도록 1~5 채택. 라벨은 UI 표시·검색·대화용 보조.

---

## 7. 카테고리 체계

문장의 **상황·도메인**을 기준으로 3개 카테고리.

| 코드 | 영역 |
|---|---|
| `DAILY` | 일상 회화, 사적 대화 |
| `BUSINESS` | 업무·이메일·미팅 |
| `TECH` | IT·개발 관련 표현 |

> 세분화·출처 등 추가 메타데이터는 운영하며 필요해진 시점에 별도 검토.
