## 1. 프로젝트 / 패키지 구조

### 1.1 아키텍처

레이어드 (Controller / Service / Repository).

### 1.2 루트 패키지

`com.tmi.core`

### 1.3 패키지 분할

**도메인 우선 → 레이어 후속**.

```
com.tmi.core
├── CoreApplication                  ← @SpringBootApplication
├── {도메인}/                          ← 도메인 단위 (sentence, answer, …)
│   ├── controller/
│   ├── service/
│   ├── repository/
│   ├── domain/                      ← 엔티티 + 값 객체
│   └── dto/                         ← request·response 통합
├── infra/{외부시스템}/                 ← Similarity 등 외부 클라이언트
└── common/                          ← Spring 설정, 상수·enum, 예외
    ├── config/
    ├── define/
    └── exception/
```

의존 방향: `controller → service → repository → domain`. `domain`은 다른 패키지에 의존하지 않음.

### 1.4 도메인 목록

| 도메인 | 책임 | 단계 |
|---|---|---|
| `sentence` | 영어 문장 출제·조회 | MVP |
| `answer` | 답안 채점 (Similarity 호출 + 응답 조립) | MVP |

이력 저장·통계·사용자 등은 MVP 이후 검토 사항으로 별도 도메인 분리 가능 (→ [01. 기획 §4.2](./01_기획.md#42-mvp-이후-검토-사항)).

## 2. 네이밍

### 2.1 기본

Java 표준 스타일 (Google Java Style 또는 IntelliJ 기본) 채택. 들여쓰기 4 spaces, 메서드·변수 camelCase.

### 2.2 계층 Suffix

| 레이어 | Suffix | 예시 |
|---|---|---|
| Controller | `Controller` | `SentenceController`, `AnswerController` |
| Service | `Service` | `SentenceService`, `AnswerService` |
| Repository | `Repository` | `SentenceRepository` |

### 2.3 DTO

- Request: `{Verb}{Domain}Request` 또는 `{Domain}{용도}Request` (예: `SubmitAnswerRequest`)
- Response: `{Domain}Response` 또는 `{Domain}{용도}Response` (예: `SentenceResponse`, `AnswerResultResponse`)

### 2.4 변수·컬럼

- Java 변수·필드: camelCase (`sentenceId`)
- DB 컬럼: snake_case (`sentence_id`) — §3.1

## 3. DB 스키마

### 3.1 테이블·컬럼 명명

- `snake_case`
- 테이블명은 **단수형** (`sentence`, `answer`)
- FK 컬럼은 `{참조 테이블}_id` (`sentence_id`)

### 3.2 PK / ID

- PK는 단일 컬럼 `id`, 타입 `varchar(36)` (순수 UUID)
- 서비스 레이어에서 `UUID.randomUUID().toString()`으로 생성 (DB default 미사용)
- **예외**: 카탈로그성 enum 값(예: `level`, `category`)은 의미 있는 식별자 그대로 (`LEVEL_3`, `DAILY`)

### 3.3 타입

| 용도 | 타입 |
|---|---|
| ID | `varchar(36)` |
| 짧은 문자열 | `varchar(N)` — 길이 명시 |
| 긴 문자열 (문장 본문 등) | `text` |
| 시각 | `timestamptz` (UTC). `timestamp` 사용 금지 |
| 정수 | `int` / `bigint` |
| 소수 (유사도 점수 등) | `numeric`. `float` / `double` 사용 금지 |
| 가변 구조 | `jsonb` |

### 3.4 enum

- PostgreSQL `enum` 타입 사용 금지
- **`varchar(N) + CHECK constraint`** 패턴

```sql
level varchar(20) NOT NULL
  CHECK (level IN ('LEVEL_1', 'LEVEL_2', 'LEVEL_3', 'LEVEL_4', 'LEVEL_5'))
```

- JPA 매핑은 `@Enumerated(EnumType.STRING)` + 같은 문자열

### 3.5 공통 컬럼

사용자 자원 테이블(`sentence` 등)에 적용.

| 컬럼 | 타입 | 비고 |
|---|---|---|
| `created_at` | `timestamptz NOT NULL DEFAULT now()` | |
| `updated_at` | `timestamptz NOT NULL DEFAULT now()` | |
| `deleted_at` | `timestamptz NULL` | NULL이면 활성 (soft delete) |

이력 테이블(예: 향후 `answer_history`)에는 `created_at`만 적용 — append-only 감사 영역.

### 3.6 Soft Delete

- 사용자 자원 테이블에 적용. `deleted_at IS NULL`이면 활성.
- 조회 쿼리는 항상 `WHERE deleted_at IS NULL`.
- 예외: 이력 테이블 — 사용자가 수정·삭제할 수 없는 append-only 감사 영역.

### 3.7 마이그레이션

- **Flyway·Liquibase 등 마이그레이션 도구 미도입** (→ [memory: project-tmi-no-flyway]).
- 스키마 변경은 [04. 데이터 모델](./04_데이터모델.md) 문서를 SSOT로 관리, 적용 방식은 04 §X 참고.

## 4. API 컨벤션

### 4.1 URL

- 버저닝: `/v{n}/...` (`application.yml`의 `rest.api.version` 치환)
- 자원 명: **복수형 + kebab-case** (`/sentences`, `/answers`)
- URL에 동사 금지. 동사형 액션은 **변경 대상**(상태·속성)을 sub-path로 표현
  - ❌ `POST /answers/{id}/finalize`
  - ✅ `PATCH /answers/{id}/status` (상태 변경)
  - ✅ `PUT /users/{id}/password` (비밀번호 변경)

### 4.2 HTTP 메서드 ↔ 응답 코드

| 메서드 | 용도 | 응답 |
|---|---|---|
| GET | 조회 | 200 |
| POST | 생성 | 201 + `Location` 헤더 |
| PUT | 전체 수정 | 200 |
| PATCH | 부분 변경 | 200 |
| DELETE | 삭제 | 204 |

### 4.3 페이징

커서 기반. `limit` 기본 20 / 최대 100.

응답:
```json
{
  "items": [ ... ],
  "page": { "cursor": "...", "nextCursor": "...", "limit": 20, "hasMore": true }
}
```

### 4.4 시간 형식

ISO 8601 UTC (`2026-05-08T10:00:00Z`).

## 5. 에러 처리

### 5.1 베이스

Spring의 `@RestControllerAdvice` 기반 글로벌 핸들러로 일원화. 응답 포맷·핸들러·다국어 메시지는 자체 컴포넌트로 구성.

작성 대상.

| 작성할 것 | 위치 | 역할 |
|---|---|---|
| `IErrorType` 인터페이스 | `common/exception/` | ErrorType enum 의 공통 계약 (`getErrorCode()`) |
| `ETmiErrorType` enum | `common/exception/` | **모듈 단일 enum** — 공통·도메인 에러 모두 통합 |
| `BaseException` | `common/exception/` | 모든 도메인 예외의 추상 부모 (HttpStatus + IErrorType + args 보유) |
| `{Domain}Exception` 클래스 | 각 도메인 패키지의 `exception/` | **외부 클래스 + 정적 중첩 클래스** — 케이스마다 중첩 클래스 1 개 |
| `GlobalExceptionHandler` | `common/exception/` | `@RestControllerAdvice` 핸들러 |
| `messages*.properties` | `src/main/resources/messages/` | 코드별 사용자 메시지 |

→ 공통 영역의 인프라(`IErrorType`, `ETmiErrorType`, `BaseException`, `GlobalExceptionHandler`)는 [99. 공통 기능](./99_공통.md) 참조.

### 5.2 응답 포맷

```
HTTP 404
{
  "error": {
    "code": "TMI.SENTENCE.NOT_FOUND",
    "message": "Sentence abc-123 was not found."
  }
}
```

- 래핑 구조 `{ "error": { code, message } }` — `GlobalExceptionHandler`가 `ErrorResponse` DTO로 직렬화.
- `code` 형식: **`TMI.{DOMAIN}.{REASON}`**.
- `message`: 영문 기본. 다국어는 `messages_{locale}.properties`로 추가.

### 5.3 ErrorType enum

- `IErrorType` 인터페이스에 `getErrorCode()` 한 메서드만.
- HttpStatus 는 enum 에 묶지 않음 — 예외 생성 시점에 명시.
- **모듈 단일 enum** `ETmiErrorType` 에 공통·도메인 에러 모두 통합. 도메인별 enum 분리 X.

```java
@RequiredArgsConstructor
@Getter
public enum ETmiErrorType implements IErrorType {

    // 공통
    INVALID_REQUEST     ("TMI.COMMON.INVALID_REQUEST"),
    INTERNAL_ERROR      ("TMI.COMMON.INTERNAL_ERROR"),
    NOT_FOUND_URL       ("TMI.COMMON.NOT_FOUND_URL"),
    METHOD_NOT_SUPPORTED("TMI.COMMON.METHOD_NOT_SUPPORTED"),
    MISSING_PARAMETER   ("TMI.COMMON.MISSING_PARAMETER"),
    BAD_REQUEST         ("TMI.COMMON.BAD_REQUEST"),

    // Sentence
    SENTENCE_NOT_FOUND  ("TMI.SENTENCE.NOT_FOUND"),

    // Similarity
    SIMILARITY_REQUEST_INVALID("TMI.SIMILARITY.REQUEST_INVALID"),
    SIMILARITY_UNAVAILABLE    ("TMI.SIMILARITY.UNAVAILABLE"),
    ;

    private final String errorCode;
}
```

### 5.4 예외 클래스 — 외부 클래스 + 정적 중첩 클래스

- 도메인별로 **외부 클래스 1 개** (`{Domain}Exception`) + 케이스마다 **정적 중첩 클래스 1 개** (예: `NotFound`, `LimitExceeded`).
- 외부 클래스는 인스턴스화 금지 — `private` 생성자.
- 각 정적 중첩 클래스는 `BaseException` 직접 상속. 생성자에서 `HttpStatus` + `IErrorType` + 가변 인자 전달.

```java
public class SentenceException {

    private SentenceException() {}

    /** 조건에 맞는 영어 문장이 없음 */
    public static class NotFound extends BaseException {
        public NotFound(String detail) {
            super(HttpStatus.NOT_FOUND, ETmiErrorType.SENTENCE_NOT_FOUND, detail);
        }
    }
}
```

서비스에서:
```
throw new SentenceException.NotFound(sentenceId);
```

`GlobalExceptionHandler` 가 `BaseException` 을 잡아 `e.getHttpStatus()` + 메시지 치환 → `ErrorResponse` 변환.

**규칙**
- 새 에러 추가 시: `ETmiErrorType` enum 상수 + 정적 중첩 클래스 + `messages.properties` 한 줄.
- **Controller 에서 try/catch 금지**, **추가 `@RestControllerAdvice` 작성 금지** (핸들러 단일화).

### 5.5 메시지

- `messages.properties`: 영문 (기본)
- `messages_ko.properties`: 한글
- `{0}`, `{1}`은 예외 생성자의 가변 인자 순서대로 치환

```properties
# messages.properties
TMI.SENTENCE.NOT_FOUND = Sentence {0} was not found.
TMI.SIMILARITY.UNAVAILABLE = Similarity scoring service is unavailable.
TMI.SIMILARITY.REQUEST_INVALID = Invalid request to similarity service.
```

### 5.6 HTTP status 선택 기준

| 상황 | status |
|---|---|
| 잘못된 요청 (validation 실패, 형식 오류) | `400 Bad Request` |
| 인증 실패 | `401 Unauthorized` |
| 권한 없음 | `403 Forbidden` |
| 리소스 미존재 | `404 Not Found` |
| 상태 충돌 (중복 / 사용 중 / 한도 초과) | `409 Conflict` |
| 서버 내부 오류 | `500 Internal Server Error` |
| 외부 시스템 호출 실패 (Similarity) | `502 Bad Gateway` 또는 `503 Service Unavailable` |

### 5.7 외부 시스템 예외 매핑

외부 시스템 호출 시 발생하는 raw HTTP/네트워크 예외는 외부 시스템 단위의 `BaseException` 자식으로 변환 후 throw. §5.4 의 외부 클래스 + 정적 중첩 클래스 패턴을 외부 시스템에도 동일하게 적용.

```java
try {
    return similarityClient.score(request);
} catch (HttpClientErrorException e) {
    throw new SimilarityException.RequestInvalid(e);
} catch (ResourceAccessException | HttpServerErrorException e) {
    throw new SimilarityException.Unavailable(e);
}
```

`SimilarityException` 도 외부 클래스 + 정적 중첩 클래스 패턴:
```java
public class SimilarityException {
    private SimilarityException() {}

    public static class RequestInvalid extends BaseException {
        public RequestInvalid(Throwable cause) {
            super(HttpStatus.BAD_GATEWAY, ETmiErrorType.SIMILARITY_REQUEST_INVALID, cause);
        }
    }

    public static class Unavailable extends BaseException {
        public Unavailable(Throwable cause) {
            super(HttpStatus.SERVICE_UNAVAILABLE, ETmiErrorType.SIMILARITY_UNAVAILABLE, cause);
        }
    }
}
```

HTTP Interface 사용 시 — Spring `RestClient` 의 기본 동작이 `HttpClientErrorException`/`HttpServerErrorException`/`ResourceAccessException` 던지므로 위 try/catch 그대로 적용.

raw HTTP 클라이언트 예외 전파 금지 (외부 응답 형식의 사용자 노출 방지).

## 6. 로깅

### 6.1 베이스

Spring Boot 기본 SLF4J + Logback 사용. 분산 추적이 필요해지면 Micrometer Tracing(Brave / OpenTelemetry) 의존성 추가로 `traceId` / `spanId` 자동 주입.

### 6.2 기본 설정

| 영역 | 설정 |
|---|---|
| 로그 라이브러리 | SLF4J + Logback (Spring Boot starter 기본) |
| 콘솔 포맷 | Spring Boot 기본 (timestamp / level / thread / logger / msg) |
| 파일 포맷 | 필요 시 `logback-spring.xml`에서 JSON encoder(예: logstash-logback-encoder) 적용 |
| 파일 회전 | 필요 시 `RollingFileAppender` + 일일 rotate + gzip |
| 분산 추적 MDC | `traceId`, `spanId` (Micrometer Tracing 활성화 시 자동) |
| 민감정보 마스킹 | 별도 라이브러리 미사용. 코드에서 토큰·비밀번호를 출력하지 않는 규칙으로 통제 |

### 6.3 프로젝트 설정 (`application.yml`)

```yaml
logging:
  level:
    root: INFO
    com.tmi.core: DEBUG   # 운영은 INFO
```

### 6.4 로거 선언

- `@Slf4j` (Lombok) 사용.
- `LoggerFactory.getLogger()` 직접 호출 금지.

```java
@Slf4j
@Service
public class SentenceService { ... }
```

### 6.5 메시지 작성 규칙

**작성:**
```java
log.info("문제 출제 요청: level={} category={}", level, category);
log.error("유사도 서버 호출 실패: sentenceId={}", sentenceId, e);
```

**출력 (Spring Boot 기본 포맷 예):**
```
2026-05-28T10:00:00.000+09:00  INFO 12345 --- [http-nio-8080-exec-3] c.t.c.s.service.SentenceService          : 문제 출제 요청: level=LEVEL_3 category=DAILY
2026-05-28T10:00:01.000+09:00 ERROR 12345 --- [http-nio-8080-exec-3] c.t.c.a.service.AnswerService           : 유사도 서버 호출 실패: sentenceId=abc-123
org.springframework.web.client.ResourceAccessException: I/O error on POST request for "http://similarity/score": Connection refused
    at ...
```

- 변수 출력은 placeholder `{}` 사용 — 문자열 `+` 연결 금지.
- ERROR 로그는 `Throwable`을 마지막 인자로 전달 → stacktrace 자동 부가.
- 메시지 언어는 한글·영문 자유.

### 6.6 로그 레벨 사용 기준

| 레벨 | 용도 |
|---|---|
| `INFO` | 주요 비즈니스 이벤트 (출제, 채점 시작·완료, 외부 호출 시작·완료) |
| `DEBUG` | 상세 파라미터·반환값 추적 (운영 비활성) |
| `WARN` | 예외 복구 시나리오, 보안 이벤트 (인증 실패 등) |
| `ERROR` | 예외 발생 — 스택트레이스 필수 (`Throwable` 마지막 인자) |

### 6.7 금지

- 민감정보 raw 로깅 (인증 토큰, API 키, 비밀번호, 외부 시크릿).
- 운영 환경 DEBUG 활성화.
